# PrintShop 知识库 7 卡体系说明

> **用途：** 统一的知识管理体系，支撑 AI 助理的专业化服务

---

## 📚 7 卡体系概览

| 卡片类型 | 核心作用 | 谁用 | 何时用 |
|---------|---------|------|--------|
| **1. 品类卡** | What - 产品是什么 | 业务/客服 | 客户询价时 |
| **2. 工艺卡** | How - 怎么做出来 | 生产/技术 | 生产准备时 |
| **3. 文件卡** | Spec - 文件规范 | 设计/技术 | 文件审核时 |
| **4. 报价卡** | Price - 多少钱 | 业务/财务 | 报价核算时 |
| **5. 交付卡** | Deliver - 怎么交付 | 生产/物流 | 订单履约时 |
| **6. 话术卡** | Say - 怎么沟通 | 业务/客服 | 客户沟通时 |
| **7. 供应商卡** | Who - 谁来做 | 采购/生产 | 供应商管理时 |

---

## 🎯 设计理念

### 核心原则

1. **结构化知识**
   - 统一格式，易于检索
   - 字段标准化，方便计算
   - 关联清晰，可追溯

2. **可计算性**
   - 报价规则可编程
   - 成本自动核算
   - 交期自动推算

3. **可维护性**
   - 版本控制
   - 更新日志
   - 责任到人

4. **AI 友好**
   - Markdown 格式
   - 结构化表格
   - 清晰的卡片间引用

## 🔗 卡片关联关系

```
品类卡 (画册装订)
   ├─→ 工艺卡 (胶装/骑马钉)
   ├─→ 文件卡 (文件规范)
   ├─→ 报价卡 (价格模型)
   ├─→ 交付卡 (交付流程)
   ├─→ 话术卡 (沟通话术)
   └─→ 供应商卡 (供应商列表)
```

**设计逻辑：**
- **品类卡**是入口，定义"是什么"
- 其他6张卡围绕品类展开，回答"怎么做"

## 🎓 使用指南

### 如何创建新卡片

**示例：创建"画册装订"品类卡**

1. **复制模板**
   ```bash
   cp templates/01-品类卡模板.md categories/画册装订/品类卡.md
   ```

2. **填充信息**
   - 根据实际业务填写字段
   - 替换示例数据
   - 保留模板结构

3. **创建关联卡片**
   ```
   categories/画册装订/
   ├── 品类卡.md
   ├── 工艺卡-胶装.md
   ├── 工艺卡-骑马钉.md
   ├── 文件卡.md
   ├── 报价卡.md
   ├── 交付卡.md
   ├── 话术卡.md
   └── 供应商卡-XX印刷.md
   ```

4. **维护关联关系**
   - 在品类卡底部添加关联链接
   - 确保双向引用

```markdown

**创建日期：** 2026-02-10  
**维护人：** 员工2

```

### 2. 供货价保密

**⚠️ 硬约束：** 
- 供货价只出现在内部卡片
- 客户侧输出前必须过滤
- AI 生成报价前检查

### 3. 数据实时性

**定期更新：**
- 报价卡：每月检查纸张价格
- 供应商卡：每季度更新合作数据
- 话术卡：根据客户反馈优化

### 4. 一致性检查

**关联卡片数据必须一致：**
- 品类卡的价格区间 = 报价卡的阶梯价
- 工艺卡的交期 = 交付卡的标准交期
- 供应商卡的报价 = 报价卡的成本

## 🔍 常见问题

**Q: 为什么是7张卡，不是更多或更少？**
A: 7张卡覆盖了图文快印业务的完整链条：
   - 产品定义（品类卡）
   - 生产流程（工艺卡+文件卡）
   - 商务条款（报价卡+交付卡+供应商卡）
   - 客户沟通（话术卡）

**Q: 卡片可以扩展吗？**
A: 可以。7卡是基础体系，可根据业务需要扩展：
   - 方案卡（Solution）：针对特定场景的完整方案
   - 案例卡（Case）：成功案例和经验总结
   - 问题卡（FAQ）：常见问题库

**Q: AI 如何使用这些卡片？**
A: OpenClaw 通过：
   1. 语义搜索找到相关卡片
   2. 提取结构化数据
   3. 执行计算逻辑（报价/交期）
   4. 生成客户回复
